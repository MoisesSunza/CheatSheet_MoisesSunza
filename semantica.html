<Semántica>

Etiquetas de Semántica

Las etiquetas semánticas se introdujeron principalmente en HTML5 para dar un mayor significado al contenido. Antes, los desarrolladores usaban mayormente <div> para todo, lo que dificultaba la lectura y el análisis del código. Ahora, con elementos como <header>, <footer>, <article> o <section>, es más fácil entender la función de cada bloque. Esto mejora la accesibilidad (lectores de pantalla identifican mejor el contenido), el SEO (los buscadores clasifican la información de manera más precisa) y la organización del sitio, facilitando la colaboración entre desarrolladores.

<header>

Representa la cabecera de un documento o sección. Normalmente contiene títulos, logotipos o menús de navegación.

  • Características:
    • Encapsula información introductoria o de navegación.
    • Puede aparecer más de una vez en la página (una por sección o artículo).
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<header>
    <h1>Mi Sitio Web</h1>
    <img src="logo.png" alt="Logo del sitio">
</header>

<nav>

Representa un bloque de enlaces de navegación principal o secundario dentro de un documento.

  • Características:
    • Normalmente contiene listas de enlaces (<ul> o <ol>).
    • Mejora la accesibilidad y SEO al indicar áreas de navegación.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<nav>
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="contacto.html">Contacto</a></li>
    </ul>
</nav>

<main>

Representa el contenido principal del documento, que es único y relevante en la página.

  • Características:
    • Solo debe aparecer una vez por página.
    • No debe incluir contenido repetitivo de cabecera, pie o navegación.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<main>
    <h2>Contenido Principal</h2>
    <p>Este es el contenido principal de la página.</p>
</main>

<section>

Define secciones temáticas de un documento, agrupando contenido relacionado, como capítulos o bloques de información.

  • Características:
    • Organiza contenido de manera lógica.
    • Puede contener títulos, párrafos, listas y otros elementos semánticos.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<section>
    <h2>Noticias</h2>
    <p>Últimas novedades sobre tecnología.</p>
</section>

<article>

Representa contenido independiente y autocontenido, como artículos, entradas de blog o noticias.

  • Características:
    • Ideal para contenido que puede ser distribuido o sindicado de forma independiente.
    • Puede contener su propio encabezado, secciones y pie de página.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<article>
    <h2>Nuevo lanzamiento</h2>
    <p>Se ha lanzado un nuevo producto al mercado.</p>
</article>

<aside>

Representa contenido relacionado indirectamente con el contenido principal, como barras laterales, notas o anuncios.

  • Características:
    • Separa información complementaria o contextual.
    • No forma parte del flujo principal del contenido.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<aside>
    <h3>Publicidad</h3>
    <p>Este es un anuncio relacionado con el contenido.</p>
</aside>

<footer>

Representa el pie de página de un documento o sección. Generalmente contiene información de contacto, derechos de autor o enlaces relacionados.

  • Características:
    • Encapsula información final de una página o sección.
    • Puede repetirse en diferentes secciones o artículos.
  • Atributos: Globales (id, class, style, data-*).

Ejemplo:

<footer>
    <p>&copy; 2025 Mi Sitio Web. Todos los derechos reservados.</p>
    <a href="contacto.html">Contacto</a>
</footer>
